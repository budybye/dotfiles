# Error Handling

## Principles

- エラーは握り潰さない。必ず処理するか、意図的に上位へ伝播させる
- システム境界（ユーザー入力、外部API、ファイルI/O）では必ずバリデーションする
- 内部コード間ではフレームワークの型保証を信頼し、過剰なバリデーションを避ける

## Patterns

### Fail-Fast
- 前提条件を関数冒頭で検証し、違反時は即座にエラーを返す
- 正常系の処理を深いネストに閉じ込めない

### Error Types
- 回復可能なエラー: Result/Option 型、try-catch で処理
- 回復不能なエラー: panic / throw で即座に停止
- ユーザー向けエラー: 技術的詳細を露出せず、操作可能なメッセージを返す

### Logging
- エラーログには原因・コンテキスト・スタックトレースを含める
- センシティブ情報（APIキー、パスワード、個人情報）をログに出力しない
- ログレベルを適切に使い分ける: error > warn > info > debug

## Anti-Patterns

- 空の catch ブロック
- エラーを文字列比較で判定する
- 例外をフロー制御に使う
- すべての例外を同一ハンドラで握り潰す
