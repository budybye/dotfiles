# Code Style

## Principles

- **KISS**: 素直で明快な解法を選ぶ。過度な抽象化を避ける
- **DRY**: 重複コードは共通関数に抽出する。ただし2回目の重複まで我慢してから抽象化する
- **YAGNI**: 現時点で必要な機能だけ実装する。将来の要件を先取りしない

## Structure

- 1ファイル 200-400行を目安。超えたら分割を検討
- 機能/ドメイン単位で整理する（ファイル種別単位ではなく）
- 高凝集・低結合を維持する

## Naming

- 変数・関数: camelCase (JS/TS), snake_case (Python/Rust/Go/C++)
- 型・クラス: PascalCase
- 定数: UPPER_SNAKE_CASE
- 名前は意図を表現する。略語は広く認知されたもののみ使用

## Immutability

- 既存オブジェクトを変更せず、新しいオブジェクトを生成する
- `const` / `let` (not `var`), `final`, `readonly` を積極的に使う
- 副作用のない純粋関数を優先する

## Quality Checklist

- [ ] 関数は1つの責務に集中しているか
- [ ] ネストは3段以下か（早期 return で浅くする）
- [ ] マジックナンバーは定数化されているか
- [ ] エラーパスが明示的に処理されているか
- [ ] コメントは「なぜ」を説明しているか（「何を」はコードで表現）
