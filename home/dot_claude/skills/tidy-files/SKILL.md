# Tidy Files - Systematic File Organization

This skill provides methodologies and tools for organizing files, directories, and codebases in a systematic and maintainable way. It covers file organization principles, directory structure best practices, code file management, and automation techniques.

## What This Skill Covers

### File Organization Principles
- Systematic approaches to categorizing and grouping files
- Naming conventions for consistency and clarity
- Directory structure design for scalability

### Codebase Management
- Code file organization within projects
- Module and component structuring
- Dependency management and file relationships

### Automation Techniques
- Script-based file organization
- Batch processing of files
- Rule-based file classification

## When to Use

- Cleaning up messy or disorganized directories
- Establishing consistent project structures
- Refactoring codebases for better organization
- Creating maintainable file hierarchies
- Implementing systematic file naming conventions
- Managing large collections of files

## Common Patterns

1. **Categorization First**: Group similar files before organizing
2. **Consistent Naming**: Apply uniform naming conventions
3. **Hierarchical Structure**: Create logical directory hierarchies
4. **Automation Where Possible**: Use scripts for repetitive tasks
5. **Documentation**: Maintain organization documentation

## Red Flags to Watch For

| Issue | Solution |
|-------|----------|
| Inconsistent naming conventions | Establish and enforce standards |
| Deeply nested directories | Flatten structure where possible |
| Mixed file types in directories | Separate by function or type |
| Poorly named files | Rename with descriptive names |
| Lack of documentation | Add README files and comments |
