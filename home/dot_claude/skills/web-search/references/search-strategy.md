# Web Search Strategy

This document outlines the methodology for effective web searching, including query formulation, source evaluation, information synthesis, and handling edge cases.

## Query Type Identification

Before formulating search queries, identify the type of information needed:

1. **Factual Queries**: Seeking specific, verifiable information (population figures, dates, definitions)
2. **Current Events**: Recent developments, news, ongoing situations
3. **Instructional**: How-to guides, tutorials, step-by-step processes
4. **Analytical**: Comparative information, pros/cons, evaluations
5. **Predictive**: Forecasts, trends, future developments

Different query types require different search strategies and source preferences.

## Query Formulation

Effective search queries are crucial for finding relevant information quickly:

### Best Practices

1. **Be Specific**: Use specific terms rather than vague ones
   - Good: "Python asyncio tutorial 2024"
   - Poor: "Python stuff"

2. **Use Quotation Marks for Exact Phrases**: When looking for an exact phrase
   - Example: "climate change effects on polar bears"

3. **Use Boolean Operators**: Combine terms effectively
   - AND: Requires both terms (often implicit)
   - OR: Either term (use parentheses: (term1 OR term2))
   - NOT or -: Exclude terms (NOT term or -term)

4. **Use Site-Specific Searches**: Limit to specific domains when appropriate
   - Example: "machine learning site:edu"

5. **Use Date Filters**: For current information
   - Many search engines support time-based filters

### Query Refinement

If initial results aren't helpful:
1. Try synonyms or related terms
2. Broaden or narrow the scope
3. Remove unnecessary qualifiers
4. Add context terms

## Source Evaluation

Not all sources are equally reliable. Evaluate sources based on:

### Authority
- Author credentials and expertise
- Publisher reputation
- Domain (.gov, .edu, .org often more reliable, but check content)

### Accuracy
- Factual correctness
- Citations and references
- Cross-referencing with other sources

### Currency
- Publication date
- Frequency of updates
- Relevance of information to current context

### Purpose
- Informative vs. persuasive intent
- Presence of bias
- Commercial interests

## Information Synthesis

Once relevant sources are found:

1. **Extract Key Points**: Identify main arguments and supporting evidence
2. **Cross-Reference**: Verify important facts across multiple sources
3. **Identify Consensus**: Note areas of agreement and disagreement
4. **Synthesize**: Combine information into coherent answers
5. **Attribute Sources**: Always credit original sources

## Common Failures

Avoid these common pitfalls:

1. **Confirmation Bias**: Only seeking information that confirms pre-existing beliefs
2. **Source Snobbery**: Dismissing useful information from less prestigious sources
3. **Information Overload**: Including too much irrelevant detail
4. **Insufficient Verification**: Accepting information without checking credibility
5. **Outdated Information**: Using old information when current data is needed

## Tool Recommendations

Depending on the context, different search tools may be appropriate:

- **General Web Search**: Google, DuckDuckGo, Bing
- **Academic Research**: Google Scholar, JSTOR, IEEE Xplore
- **News**: Google News, Reuters, Associated Press
- **Technical Documentation**: Official documentation sites, Stack Overflow
- **Real-time Information**: Twitter, Reddit (with caution)

Always consider the nature of the information needed when selecting search tools.

## Handling Search Failures

When searches return no results or poor quality results:

1. **Query Refinement**:
   - Check for typos or ambiguous terms
   - Try synonyms or related terms
   - Simplify complex queries
   - Break multi-part questions into simpler ones

2. **Alternative Approaches**:
   - Use different search engines
   - Try site-specific searches
   - Look for related topics that might contain relevant information
   - Consider if the query is too recent or too obscure

3. **User Communication**:
   - Explain the difficulty clearly
   - Suggest alternative approaches
   - Ask for clarification if the query is ambiguous
   - Provide partial information when possible
