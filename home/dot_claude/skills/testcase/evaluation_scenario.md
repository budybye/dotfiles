# Evaluation Scenario for Testcase Skill

## Scenario
You are a software developer working on a new web application that allows users to manage a personal todo list. The application has the following features:
- User registration and login
- Creating, reading, updating, and deleting todos
- Marking todos as complete/incomplete
- Filtering todos by status (all, active, completed)
- Due date tracking for todos

The application is built with a modern JavaScript framework (like React or Vue) with a backend API. You need to use the `/testcase` skill to plan what to test before writing any test code.

## Requirement Checklist
1. [critical] Identify functional, integration, and non-functional test cases for the todo application
2. [critical] Properly classify and prioritize test cases using the P0-P3 system
3. Generate a properly formatted test-case list document with TC-IDs
4. Create a roadmap/progress document with implementation order
5. Include appropriate test details (preconditions, inputs, expected outputs)
6. Follow the established conventions for TC-ID formatting and document structure

## Task
Using the `/testcase` skill, create a comprehensive test plan for the todo application described above. Follow the guidelines and frameworks provided in the skill and ensure your plan addresses all the requirements in the checklist above.

## Expected Deliverable
Two structured documents:
1. A test-case list (`tests/testlist.md`) that outlines all identified test cases with proper classification, prioritization, and details
2. A roadmap document (`tests/TDD.md`) that provides implementation order and TDD guidance
