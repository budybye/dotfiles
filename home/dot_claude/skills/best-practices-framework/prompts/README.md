# Prompt Templates for Best Practices Framework

This directory contains prompt templates that can be used with the best-practices-framework skill. These templates help structure common tasks and ensure consistent application of the framework's methodologies.

## Available Templates

### `/evaluate-stack` - Technology Stack Evaluation
Evaluate technology stacks for cross-domain decisions with structured analysis.

**Usage**: `/evaluate-stack web-application "high-performance" "scalable-architecture"`

### `/design-architecture` - Architecture Design
Design system architectures using best practices patterns.

**Usage**: `/design-architecture "e-commerce-platform" "microservices" "high-availability"`

### `/assess-practices` - Development Practice Assessment
Assess current development practices and recommend improvements.

**Usage**: `/assess-practices 10 "React-Node.js" "code-review-process"`

### `/integrate-tools` - Tool Integration Planning
Plan integration of new tools with existing workflows.

**Usage**: `/integrate-tools "Datadog" "Jenkins-GitHub" "production"`

### `/test-empirical` - Empirical Skill Testing
Test the skill's effectiveness with empirical scenarios.

**Usage**: `/test-empirical best-practices-framework/references/empirical_evaluation_scenarios.md`

## How to Use

1. Copy these templates to your pi prompt directory (typically `~/.pi/agent/prompts/`)
2. Restart pi or reload prompt templates
3. Type `/` in the editor to see available templates
4. Select a template and provide the required arguments

## Customization

Feel free to modify these templates to better suit your specific workflows and requirements. The templates are designed to be starting points that can be adapted to your particular context.
