# Test Skill for Evaluation (Improved)

This is an enhanced version of the test skill with better frameworks for problem-solving and decision-making.

## Usage

To use this skill, follow these steps:

1. Identify the task you want to accomplish
2. Apply the appropriate method from the frameworks below
3. Review and refine your approach as needed

## Frameworks

### Architecture Planning Framework
1. Identify system components and their responsibilities
2. Define data flow between components
3. Specify interfaces and APIs
4. Consider scalability and performance requirements
5. Plan for monitoring and error handling

### Approach Comparison Framework
1. List all viable approaches
2. Define evaluation criteria (e.g., complexity, performance, security)
3. Score each approach against the criteria
4. Identify tradeoffs and limitations
5. Select the most appropriate approach

### Security Analysis Framework
1. Identify potential threat vectors
2. Apply relevant security principles (least privilege, defense in depth, etc.)
3. List specific mitigation strategies
4. Consider compliance requirements
5. Plan for security testing

## Examples

Example 1: Authentication System Design
- Components: User service, Auth service, Database
- Data flow: Registration → User creation → Login → Token generation
- Approaches: Session-based vs JWT tokens
- Security: Password hashing, rate limiting, secure token storage

Example 2: API Design
- Components: Resource endpoints, Authentication layer, Business logic
- Interfaces: RESTful APIs with JSON payloads
- Security: OAuth2, Input validation, CORS policies
