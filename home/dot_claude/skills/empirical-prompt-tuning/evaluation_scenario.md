# Evaluation Scenario for Test Skill

## Scenario
You are a software developer tasked with implementing a new feature for a web application. The feature requires adding a user authentication system with login and registration capabilities. You need to use the test-skill-for-evaluation to guide your approach.

## Requirement Checklist
1. [critical] Identify at least 3 key considerations for implementing the authentication system
2. [critical] Provide a high-level architecture plan for the authentication feature
3. Evaluate at least 2 different authentication approaches (e.g., session-based vs token-based)
4. List potential security concerns and mitigation strategies
5. Outline the implementation steps needed to build the feature

## Task
Using the test-skill-for-evaluation, create a plan for implementing the user authentication system. Follow the guidelines provided in the skill and ensure your plan addresses all the requirements in the checklist above.

## Expected Deliverable
A structured plan document that outlines:
- Key considerations for the authentication system
- High-level architecture
- Comparison of different authentication approaches
- Security considerations
- Implementation steps
