# Go テンプレートの `-`（トリム）ガイド

chezmoi は Go の `text/template` を使用します。`-` はアクション前後の空白を削除するための記号です。

## 仕様（公式）

出典: [text/template - Text and spaces](https://pkg.go.dev/text/template#hdr-Text_and_spaces)

| 記法  | 効果                                                   |
| ----- | ------------------------------------------------------ |
| `{{-` | アクション**直前**の空白（スペース・タブ・改行）を削除 |
| `-}}` | アクション**直後**の空白を削除                         |

**重要**: `-` の直後にスペースが必要です。

- `{{- 3}}` → 直前の空白を削除してから `3` を出力
- `{{-3}}` → 数値 `-3` として解釈される（意図と異なる）

## 統一ルール

### 1. 制御構造（if/else/end, range/end）で行全体がテンプレートのみのとき

**目的**: 生成ファイルに不要な空行を残さない。

| 状況                        | 推奨                            | 例                           |
| --------------------------- | ------------------------------- | ---------------------------- |
| 行全体が `{{ if }}` のみ    | `{{-` と `-}}` を両方使う       | `{{- if eq .os "darwin" -}}` |
| 行全体が `{{ end }}` のみ   | 末尾の `-}}` で次行の空白を削除 | `{{ end -}}`                 |
| 行全体が `{{ range }}` のみ | 同様                            | `{{- range .items -}}`       |

### 2. インラインの値出力（`{{ .foo }}`）

**目的**: 値の前後のスペースをそのまま残す。

| 状況       | 推奨            | 例                            |
| ---------- | --------------- | ----------------------------- |
| 文中の値   | `-` を使わない  | `name = {{ .name \| quote }}` |
| 値のみの行 | 通常は `-` 不要 | `{{ .chezmoi.sourceDir }}`    |

### 3. コメント（`{{/* ... */}}`）

| 状況                           | 推奨                | 例                   |
| ------------------------------ | ------------------- | -------------------- |
| コメント前後の空白も削除したい | `{{- /* ... */ -}}` | `{{- /* 説明 */ -}}` |

## ファイル種別ごとの方針

### TOML（.chezmoi.toml.tmpl など）

- 空行が増えると見た目が崩れる
- 制御構造は `{{-` / `-}}` で囲む
- 例: `.chezmoi.toml.tmpl` の if/else ブロック

### Shell スクリプト（.sh.tmpl）

- 空行はシェルではほぼ無害
- ただし `range` でループするときは `-` を使うと出力がきれいになる
- 例: `run_onchange_after_vscode.sh.tmpl` の `{{ range .packages.extensions -}}`

### 構造化出力（authorized_keys, config など）

- 余計な空行を避けたい
- `{{-` / `-}}` でトリムする
- 例: `authorized_keys.tmpl`, `dot_ssh/config.tmpl`

## 注意点

1. **`{{-` の直後にスペース**
    - `{{- if` ✅
    - `{{-if` ⚠️（`if` の場合は動くが、`{{-3}}` は `-3` と解釈される）

2. **ドットの前のスペース**
    - `{{ .foo }}` ✅（`.foo` はドット変数）
    - `{{.foo}}` ⚠️（パースは通るが、可読性が低い）

3. **複数行ブロック**
    - ブロック内のインデントはそのまま出力される
    - ブロックの開始・終了行だけトリムすれば十分なことが多い

## 現在のプロジェクトでの使用例

| パターン               | 使用例                                                                     |
| ---------------------- | -------------------------------------------------------------------------- |
| 変数代入・制御のみの行 | `.chezmoi.toml.tmpl` の `{{- $name := "budybye" -}}`                       |
| 条件付きブロック       | `dot_ssh/config.tmpl` の `{{- if eq .chezmoi.os "darwin" }}`               |
| range の空行削除       | `run_onchange_after_vscode.sh.tmpl` の `{{ range .packages.extensions -}}` |
| インライン値           | `{{ .chezmoi.sourceDir }}`, `{{ .name \| quote }}`                         |

## 検証方法

### 1. chezmoi execute-template で確認

```bash
# 単一ファイルのレンダリング結果を確認
chezmoi execute-template < home/.chezmoi.toml.tmpl

# 変数付きで実行（dry-run 相当）
chezmoi execute-template --promptString name=test --promptString email=test@example.com < home/private_dot_config/git/user.conf.tmpl
```

### 2. chezmoi verify で全体確認

```bash
# 設定・テンプレート・スクリプトの整合性を検証
chezmoi verify
```

### 3. chezmoi diff で確認

```bash
# 適用後の差分を確認（テンプレートの出力が正しいか）
chezmoi diff
```

### 4. 構文チェック

- `{{.` のスペース欠如: `{{.foo}}` は `{{ .foo }}` に
- `{{-3` の誤解釈: 数値 -3 と解釈される。トリム時は `{{- 3}}` とスペースを入れる
- 未定義変数: 存在しないキー参照は空文字になる。`{{ .foo }}` で `.foo` が nil のときは何も出力されない

## ベストプラクティス

### 1. JSON 出力では必ず `| quote` を使う

JSON の文字列値はダブルクォートで囲む必要がある。

| 誤                    | 正                             |
| --------------------- | ------------------------------ |
| `"key": {{ .value }}` | `"key": {{ .value \| quote }}` |

出力例: `"key": https://example.com`（不正）→ `"key": "https://example.com"`（正）

### 2. シェルに渡す値は `| quote` を検討

パスやユーザー入力が含まれる値は、スペースや特殊文字でシェルが誤解釈する可能性がある。

```bash
# パスにスペースがある場合に安全
path="{{ .path | quote }}"
```

### 3. 制御構造の一貫性

- TOML/JSON: 制御行は `{{-` / `-}}` で囲み、余計な空行を出さない
- Shell: `range` の末尾に `-}}` を付けてループ間の空行を減らす

### 4. コメントの配置

- 行末コメント: `{{- $x := 1 -}}{{/* 説明 */}}`
- 行全体コメント: `{{/* 説明 */}}` のみの行は出力に影響しない

### 5. 存在チェック

キーが存在するか不明なときは `hasKey` を使う。

```go
{{- if hasKey . "optionalKey" }}
optionalKey = {{ index . "optionalKey" | quote }}
{{- end }}
```

## よくある落とし穴

| 落とし穴              | 原因                                 | 対処                            |
| --------------------- | ------------------------------------ | ------------------------------- |
| `{{-3}}` が -3 と解釈 | `-` の直後にスペースがない           | `{{- 3}}` とする                |
| JSON が不正           | 文字列値に `\| quote` を付けていない | 文字列は必ず `\| quote`         |
| 空行が増える          | 制御構造の前後に改行がある           | `{{-` / `-}}` でトリム          |
| 変数が空              | 未定義または nil                     | `hasKey` やデフォルト値でガード |

## クイックリファレンス

```txt
{{- if ... -}}    直前・直後の空白を削除
{{- if ... }}     直前のみ削除
{{ if ... -}}     直後のみ削除
{{ if ... }}      削除なし（そのまま出力）
{{ .foo }}        値出力（通常は - 不要）
{{ .foo | quote }} JSON/シェル用にクォート
```
